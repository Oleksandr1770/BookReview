Chapter 1. Don’t make me think!

  • Don’t make a user stop and think about your clever or marketing-induced names. Even if there is a need to use specific terminology, try to keep them between “Obvious to everybody” and “Truly obscure”.
  • Inform the user about clickable elements. Try to make them obvious.
  • User should not spend time on thinking what would happen if he clicks the button.
  • “No question marks. No mental chatter. And no errors.” (Krug, 2014)
  • Using a web site that doesn’t make a user think about unimportant things looks more attractive.

Chapter 2. How we really use the Web

  • People usually do not spend time on reading all texts to decide which link to click next. Instead of reading users scan (or skim) them because they know what they are looking for.
  • Most of the time users choose the first reasonable option even if it does not fully satisfice the user’s needs.
  • Most of the users do not read instructions.
  • Users should understand how to use things even if they have no idea about how they work.

Chapter 3. Billboard Design 101

  • Try to be creative but make sure that your web page is clear and self-explanatory.
  • Stick to the existing Web conventions. Use standardised appearance and location.
  • Use metaphors to save user’s time.
  • Each page should have a clear visual hierarchy where things are related logically and visually.
  • Divide pages into clearly defined blocks. This helps the user scan the page faster.
  • Avoid visual noise. Get read of anything that is not valuable and annoys users.
  • Structured text is easier to read. Use headings, fonts, lists, highlights to separate the ideas and make your text more scannable.

Chapter 4. Animal, Vegetable or Mineral?

  • Do not give the user difficult choices and questions. The more time a user spends on thinking, the higher chance that he leaves the web site.
  • Provide assistance only when it is required.

Chapter 5. Omit needless words

  • Remove the words that no one is going to read. It reduces the noise level and makes your page more scannable.
  • Avoid happy talks. User has a goal to find necessary information, not to read all texts.
  • If your website should not need instructions. Make pages as obvious as they can be. In case the instructions are necessary, provide a bare minimum of tips.

Chapter 6. Street signs and Breadcrumbs

  • Keep the user informed about his current location. But use a minimal version of persistent navigation for pages with forms.
  • Navigation bar should have the same location.
  • The Site ID and logo should be on every page to keep the user informed that he is still on this Site.
  • The Sections (links to the main sections) and the Utilities (links to the additional functionality) help users to navigate through the Website.
  • Persistent navigation should contain a button or a link to the home page.
  • There are two types of users: those who prefer searching and those who prefer browsing. Bring a search box to your navigation bar. It should be simple: a box, a button and the word “Search”.
  • Don’t forget about multi-level navigation. Users should spend equal time on navigation for both low-level pages and high-level pages.
  • Every page must have a name that matches the content.
  • Use breadcrumbs (location indicators). Place them at the top, use dividers to split the levels, boldface the last item.
  • Be fancy and save user’s time. Use tabs for navigation and division of the sections.

Chapter 7. The Big Bang Theory of Web Design

  • You cannot put all things to a Home page. The compromises should be worked out and an inevitable information should be removed.
  • The Home page’s main goal is to provide information of what the site is about. The Home page should contain the tagline, the Welcome blurb and the “Learn more”. These three things help users guess what the site is about.

Chapter 8. “The Farmer and the Cowman Should Be Friends”

  • There is no situation when everybody likes the page. Focus on the majority.
  • There is no average user, all Web users are unique.

Chapter 9. Usability testing on 10 cents a day

  • Focus groups are not usability tests. The main difference is that in focus groups you get users’ opinions about products while in usability tests you watch people interact with things.
  • Ask other people to test your Web site.
  • Testing is not a big deal. Use simple tests to see if you can make your Web site greater.
  • Spend one morning a month on usability testing. The ideal number of users is three. Find users from your target audience.
  • The more team members observe the testing process, the better.
  • Choose the tasks depending on what is available for testing.
  • Fix the most serious usability problems first.

Chapter 10. Mobile: It’s not just a city in Alabama anymore

  • Start designing the mobile version by choosing the most important features.
  • Prioritize the features. Things that are frequently used should be at the front.
  • If you don’t create a responsive design, let the users zoom the content.
  • Always provide a link to a full Web site.
  • Affordances should be noticeable and should allow users to do their tasks.
  • Create delightful apps that are useful.
  • If you implement innovative things, try to provide clear instructions to make user understand how to use those features.
  • Create memorable apps. User should remember how to control your app.
  • Test usability. Use cameras to see what is happening on the screen during the test.

Chapter 11. Usability as common courtesy

Things that descrease goodwill:

  • Hiding information
  • Making the users guess how to fill the forms
  • Asking aditional personal information
  • Making a disorganized and unprofessional website

Things that increase goodwill:

  • Bringing to front and making the main features easy and obvious
  • Providing the information that was requested
  • Putting the most frequently asked questions in the FAQ section
  • Helping the user track his activity
  • Making printer-friendly pages
  • Apologize when you cannot provide support to the user

Chapter 12. Accessibility and you

  • Create an accessible Web site because it makes people’s lives better.
  • The best way to start improving your Website is to test it for the accessibility issues.
